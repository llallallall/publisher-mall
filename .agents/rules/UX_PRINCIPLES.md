# 🎨 데브아울스 프로젝트 UI/UX 원칙

이 문서는 데브아울스 표준 행동 강령에 따라 모든 UI/UX 설계 및 개발 시 준수해야 할 핵심 지침입니다.

## 🏆 핵심 원칙: 사용자 주권 (User Sovereignty)

시스템은 일방적으로 사용자의 컨텍스트를 전환하지 않으며, 모든 프로세스 진행 전 사용자에게 충분한 설명과 의사결정의 기회를 제공해야 한다.

### 1. 강제 전환 금지 (No Forced Redirection)
- 로그인이 필요한 기능(결제, 자료 다운로드 등) 접근 시 사용자를 즉시 로그인 페이지로 튕겨내지 않는다.
- 대신 **정보 제공 모달(Information Modal)**을 띄워 현재 상황을 설명하고, 사용자가 로그인 여부를 스스로 결정하게 하라.

### 2. 선 설명 후 실행 (Explain before Action)
- 데이터 삭제, 대량의 통신 발생, 유료 서비스 접근 등 중요한 동작 수행 전에는 반드시 해당 동작의 의미와 결과(Impact)를 사용자에게 명확히 고지한다.

### 3. 상태 투명성 (Status Transparency)
- 장바구니 담기, 회원 가입, 결제 등 비동기 처리 시 시스템이 "작고 있는 중"임을 시각적으로 100ms 이내에 즉각 반응하여 보여주어야 한다.

### 4. 뒤로가기 보존 (Preserve Navigation Context)
- 사용자의 의사결정 과정에서 발생한 모달이나 팝업은 브라우저의 '뒤로가기' 또는 'ESC' 키로 언제든 안전하게 닫을 수 있어야 하며, 이전 입력 상태를 최대한 보존한다.

---

*이 원칙은 항시 적용되며, 어떠한 기능 구현보다 우선한다.*
